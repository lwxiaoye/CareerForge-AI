# CareerForge-AI 产品需求文档

> 面向高校学生的 AI 就业辅助平台
>
> 最后更新：2026-06-22

---

## 1. 产品定位

CareerForge 是一个**面向高校学生**的 AI 就业辅助平台，帮助学生从「简历制作 → 简历优化 → 模拟面试 → 面试复盘」全流程提升求职竞争力。

平台包含两个角色：

| 角色 | 谁用 | 怎么登录 |
|------|------|---------|
| **学生** | 高校在读/应届毕业生 | 邮箱验证码注册 → 邮箱+密码登录 |
| **管理员** | 高校就业指导中心老师 / 平台运营 | 账号+密码登录（系统初始化时自动创建） |

## 2. 用户故事与功能需求

### 2.1 学生端

#### 2.1.1 AI 简历助手

**核心价值**：学生用自然语言描述自己的经历，AI 帮他生成一份专业简历，并支持反复优化。

| 编号 | 用户故事 | 优先级 |
|------|---------|--------|
| S-R01 | 作为学生，我想跟 AI 对话描述我的经历，让它帮我生成一份简历，这样我不用从零开始写 | P0 |
| S-R02 | 作为学生，我想上传已有的简历 PDF，让 AI 基于真实内容帮我优化，而不是凭空编造 | P0 |
| S-R03 | 作为学生，我想让 AI 根据目标岗位的 JD 定制简历，突出相关经验 | P0 |
| S-R04 | 作为学生，我想把优化好的简历导出为 PDF 下载 | P0 |
| S-R05 | 作为学生，我想在修改简历前看到 AI 的修改预告，修改后看到修改总结 | P1 |
| S-R06 | 作为学生，我想撤销 AI 的修改，回到之前的版本 | P1 |
| S-R07 | 作为学生，我想让 AI 记住我的偏好（比如「不要写自我评价」），后续对话自动遵守 | P1 |
| S-R08 | 作为学生，我想看到 AI 调用了哪些工具（读简历、查档案、生成 PDF 等），了解它在做什么 | P1 |
| S-R09 | 作为学生，我想选择思考程度（自动/低/中/高），让 AI 根据任务复杂度调整回复深度 | P1 |

**业务规则**：
- AI 修改简历前必须先读取真实简历内容，**禁止凭对话记忆编造经历**
- 每次写操作前自动保存快照，最多保留 20 个历史版本，支持一键撤销
- AI 记住的约束/事实对用户可见、可删除
- 流式输出：AI 边思考边打字，工具调用过程以动画胶囊实时展示

#### 2.1.2 AI 面试官

**核心价值**：AI 扮演专业面试官，根据学生的简历和目标岗位进行一对一模拟面试，逐轮提问+评分+追问，最后生成面试报告。

| 编号 | 用户故事 | 优先级 |
|------|---------|--------|
| S-I01 | 作为学生，我想选择面试类型（通用/技术/HR/压力面试）开始一场模拟面试 | P0 |
| S-I02 | 作为学生，我想用文字回答面试问题 | P0 |
| S-I03 | 作为学生，我想用语音回答面试问题（录音转文字） | P1 |
| S-I04 | 作为学生，我想看到面试官根据我的回答进行追问，而不是照本宣科 | P0 |
| S-I05 | 作为学生，我想在面试结束后看到详细的面试报告（各维度评分、表现分析、改进建议） | P0 |
| S-I06 | 作为学生，我想查看历史面试记录和报告 | P1 |
| S-I07 | 作为学生，我想在面试过程中看到实时进度（当前阶段、已进行轮次） | P1 |

**面试阶段**（状态机驱动）：
1. **开场** — 确认目标岗位与面试类型
2. **自我介绍** — 考察自我认知和表达结构
3. **简历深挖** — 验证项目真实性、个人贡献和量化结果
4. **核心技术** — 考察岗位必备技术深度
5. **场景题** — 考察系统设计、业务理解和问题拆解能力
6. **压力追问** — 考察抗压能力、证据意识和诚实度
7. **反问环节** — 考察对岗位和公司的思考深度
8. **收束复盘** — 总结表现，给出改进方向

**业务规则**：
- 面试官的问题必须基于学生的真实简历内容（简历锚点提取）
- 每轮回答后给出评分（6 个维度：技术准确性、项目证据、问题解决、沟通表达、岗位匹配、抗压能力）
- 支持幂等提交（同一道题重复提交不会产生重复记录）
- 面试过程中实时推送 SSE 事件（阶段切换、评分、追问生成等）

## 3. 非功能需求

| 类别 | 要求 |
|------|------|
| **多租户隔离** | 所有数据按 `tenant_id` 隔离，新查询漏了 tenant_id = 跨租户数据泄漏 |
| **安全** | API Key 使用 Fernet 真加密存储；JWT 双 Token + Redis 吊销名单；登录限流（5 次/15 分钟锁定）；IP 级请求限流（200 rps/60s） |
| **数据一致性** | 数据库零外键约束（MySQL 设计要求），一致性靠应用层保证；软删除（`is_deleted`），不物理删除业务数据 |
| **流式体验** | 对话类功能全部 SSE 流式输出，边思考边打字，不缓冲 |
| **可撤销** | 简历修改前自动保存快照，支持撤销 |
| **PDF 导出** | 简历 PDF 必须在任意查看器下中文正常显示（内嵌 CJK 字体） |
| **部署** | Docker Compose 一键部署（MySQL + Redis + 后端 + 前端 + Nginx） |
