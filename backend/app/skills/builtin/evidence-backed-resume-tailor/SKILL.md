# 订制简历（事实约束 + 高质量写作）

## 适用场景

用户要求根据目标岗位 JD：

- 生成或订制针对该岗位的简历；
- 优化或改写已有简历；
- 调整项目、实习、技能和自我评价的重点；
- 分析 ATS 关键词和岗位匹配缺口；
- 针对转行、应届生、换赛道等情况定制内容策略。

---

## 不可违反的事实边界

1. **不得新增**事实来源中不存在的公司、岗位、项目、学校、专业、证书、奖项、职责、技术栈、方法、客户、用户规模、团队规模、时间、数字、比例、排名或业务成果。
2. **不得把 JD 中的技能直接写成学生技能**。只有事实来源中明确出现时，才能放入简历；否则只能列入"能力缺口"。
3. **不得升级表述**：不得把"参与"改成"主导"，不得把"了解"改成"熟练"，不得扩大职责、范围或影响。
4. **不得伪造指标**。没有数字就保持无数字表达，或向用户追问真实数据。
5. **空字段和空数组**表示尚未填写，不得为了完整度自动补齐。
6. **可以修改**目标职位、内容顺序、标题、格式、动作表达和同义词，但这些修改不能改变个人事实。

---

## 执行流程

### 第 1 步：获取并整理事实清单

先调用 `query_student_profile` 读取学生档案。若优化已有简历，还需调用 `read_resume`，或使用本轮上传文件中的内容。

将可用事实按模块整理，每项必须能指向原始字段：

| 模块 | 包含内容 |
|------|----------|
| `BASIC` | 姓名、联系方式、求职意向 |
| `EDU` | 学校、专业、学历、时间、GPA、荣誉、原始描述 |
| `WORK` | 公司、岗位、时间、原始职责与成果 |
| `PROJECT` | 项目名、角色、时间、原始描述与明确技术 |
| `SKILL` | 用户明确填写的技能（编程语言、工具、框架等） |
| `OTHER` | 证书、荣誉、发表、志愿经历 |

不能确定来源的内容标记为 `UNVERIFIED`，不得写入简历。

---

### 第 2 步：深度分析 JD 优先级

从 JD 中提取并分级：

- `P0`：硬性门槛（学历、必须技能、实习年限等），缺失则在差距分析中明确提示；
- `P1`：核心职责、高频技能、关键业务场景，简历重点覆盖；
- `P2`：加分项、软技能、行业偏好，有证据则写入，无证据不补造。

同时注意：
- 记录 JD 中**反复出现的关键词**（通常是 ATS 高权重词）；
- 识别公司文化信号（如"快速迭代""数据驱动"），用于自我评价角度；
- 注意岗位级别（实习/初级/中级/高级），决定简历篇幅和深度。

---

### 第 3 步：构建证据匹配矩阵

逐项将 JD 要求分类：

| 状态 | 含义 | 处理方式 |
|------|------|----------|
| `SUPPORTED` | 事实来源有直接证据 | 重点突出，可用 JD 同义词强化 |
| `PARTIAL` | 只有相邻或可迁移经验 | 准确描述迁移关系，不能声称已具备 |
| `GAP` | 没有证据 | 不进入简历正文，列入差距分析 |
| `NEEDS_CONFIRMATION` | 材料可能包含但信息不足 | 向用户追问真实数据再继续 |

---

### 第 4 步：高质量写作输出

**通用写作标准（每条经历/项目描述都必须遵守）：**

- 每条要点必须以**强动作动词开头**：
  - 优选词：主导、设计、实现、优化、搭建、研发、重构、部署、分析、建立、推动、领导、简化、提升……
  - 禁用词：参与、协助、了解、接触、学习（除非是培训类职责）；
- 尽量采用 **SAR 结构**：【行动 + 方法/工具】→【结果/影响】；如有背景数字（团队规模、系统量级）则前置；
- **保留所有原始数字**（性能提升百分比、用户量、代码行数、团队人数、金额），不得修改；
- 没有数字时，用规模描述代替空洞形容词（"服务百万级 DAU" 优于 "大规模系统"）；
- 每条要点字数控制在 15-40 字，避免过长。

**各模块写作规范：**

**自我评价 / 职业摘要（3-4 句）：**
1. 起句：`{工作年限/在校年级} + {核心方向}专业/工程师，具备 {P1 技能组合}`；
2. 主句：突出 1-2 个与 JD 高度匹配的真实成就或能力；
3. 结句：与 JD 文化关键词呼应（"持续学习""数据驱动"等），仅在档案中有据时使用。

**技能区：**
- 按 JD 相关度分组排序，最相关的放最前；
- 同时保留缩写和全称（如"SQL（结构化查询语言）"）；
- 只写能在经历中找到使用证据的技能。

**经历/项目顺序：**
- 按与 JD 的相关度排序（不强制按时间），最相关的放最前；
- 同一条经历下，最相关的要点排在最前。

---

### 第 5 步：保存前自检

调用 `generate_resume_data`、`optimize_resume_data` 或 `update_resume_data` 前逐项检查：

- [ ] 每个公司、岗位、项目、学校和技能是否有事实来源；
- [ ] 每个数字、百分比、日期是否与来源完全一致；
- [ ] 是否把 JD 关键词误写成个人能力；
- [ ] 是否改变了职责级别或扩大了影响；
- [ ] 是否遗漏与 P0/P1 高相关的真实证据；
- [ ] 空模块是否仍保持为空；
- [ ] 每条要点是否以强动词开头；
- [ ] 是否保留了原始数字。

任何一项无法通过时，不调用保存工具；先告诉用户缺少什么，并请求补充或确认。

---

### 第 6 步：差距分析与建议输出

保存完成后，在对话中额外输出以下内容（简洁呈现，不超过 10 行）：

**1. 岗位匹配优势（2-3 条）**
> 候选人哪些真实经历与 JD P1 要求高度吻合，面试时可重点阐述。

**2. 能力缺口（如有）**
> 哪些 P0/P1 要求在档案中没有证据；建议通过课程、项目或补充说明弥补。

**3. 可量化但未提供数字的项目（如有）**
> 请用户补充具体数字，下次优化时可加入。

**4. 封面信/面试准备提示（1-2 句）**
> 哪个经历最能打动该岗位的面试官，建议在封面信中展开。

---

## 特殊候选人策略

**应届生 / 实习生：**
- 教育模块前置（含 GPA ≥ 3.5、相关课程、竞赛奖项）；
- 重点放在项目经历和校内实习，不要为了填充而拉长描述；
- 自我评价强调学习能力和成长潜力（需有据可依）。

**转行求职者：**
- 使用混合格式：先放"核心能力"区（迁移技能），再列工作经历；
- 每段经历中突出可迁移部分，用 PARTIAL 策略准确表达；
- 自我评价说明转行动机（仅基于学生真实说法，不编造）。

**有大量工作经验者（10 年以上）：**
- 只保留与 JD 最相关的近 5-7 年经历；
- 早期经历可合并为一行（时间段 + 职位 + 公司）；
- 自我评价突出战略/领导层面影响。

---

## 对话输出规范

向用户展示（按顺序）：

1. 已读取哪些事实模块（一句话）；
2. JD 的 3-5 个核心要求（P0/P1）；
3. 匹配状态摘要（几条 SUPPORTED / 几条 GAP）；
4. 将重点调整哪些真实内容（1-3 点说明）；
5. 调用保存工具，返回 editor_url 链接；
6. 差距分析与建议（见第 6 步）。

不要输出隐藏推理、工具原始 JSON、手机号等隐私信息。

## 改写范例（Before → After）

### 技术岗

| Before（档案原文） | After（优化后） | 改写要点 |
|--------------------|-----------------|----------|
| 参与了一个电商项目，负责后端 | 主导订单模块重构，将核心接口响应时间从 800ms 降至 200ms，支撑日均 10 万笔交易 | 强动词开头 + 量化结果 + 业务规模 |
| 用 Python 写了一些脚本 | 基于 Python + Pandas 构建自动化数据清洗流水线，处理效率提升 5 倍 | 技术栈具体化 + 量化提升 |
| 负责数据库相关工作 | 设计并实施 MySQL 分库分表方案，支撑千万级用户数据存储，查询延迟降低 60% | 架构层面描述 + 量化规模和效果 |
| 了解微服务架构 | 基于 Spring Cloud 搭建微服务架构，拆分 5 个核心服务，部署至 K8s 集群 | 技术栈明确 + 量化拆分规模 |

### 运营/市场岗

| Before | After | 改写要点 |
|--------|-------|----------|
| 帮忙做了公众号运营 | 独立运营校招公众号，3 个月内粉丝从 200 增长至 5000，单篇最高阅读量 1.2 万 | 量化增长 + 具体指标 |
| 参与了一次线下活动策划 | 协调 5 人团队策划校园招聘会，吸引 200+ 学生参与，收集 80 份有效简历 | 规模量化 + 可交付成果 |
| 协助市场推广工作 | 负责 3 个社交媒体渠道内容运营，月均触达用户 5 万+，获客成本降低 30% | 渠道明确 + 量化效果 |

### 通用原则

1. **没有数字时**：用规模描述替代（"服务百万级 DAU"优于"大规模系统"）
2. **无法量化时**：描述影响范围（"覆盖 3 个业务线"、"被 2 个团队复用"）
3. **永远不要编造数字**：没有就保持无数字表达，或在差距分析中提示学生补充
