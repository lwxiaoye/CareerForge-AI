# PaddleOCR 简历识别

## 适用场景
当学生上传 PDF 简历，且原文件存在扫描页、双栏排版、图片化文字或复制文本质量差的问题时，优先使用本技能。

## 处理原则
1. PDF 一律优先走 OCR / 版面识别，先拿稳定文本，再考虑纯文字抽取。
2. OCR 结果不足时，才回退到普通文本抽取，避免整页乱码直接进入首问。
3. 锚点如果只是半句描述、能力短语或被截断片段，禁止直接拿来当项目名发问。
4. 无法确认真实项目名时，首问改成更保守的问法，只要求候选人介绍一段最相关的真实项目/实习/工作经历。

## 推荐输入
- 文件名
- 文件类型
- PDF 页图片列表
- 目标岗位名称
- 候选人简历来源说明

## 推荐输出
- extracted_text: 清洗后的简历文本
- structured_resume: 结构化项目、工作、实习、教育信息
- anchors: 候选锚点列表
- best_opening_anchor: 可安全用于首问的锚点；不安全时返回空
- ocr_attempts: 本次 OCR 的来源、状态、页数与失败原因
- fallback_reason: 为什么回退到纯文字抽取或保守首问

## 成功标准
- 真实项目名优先于“半句能力描述”
- OCR 不可用时也能平稳回退
- 首问不会把片段文本误说成候选人真实项目
